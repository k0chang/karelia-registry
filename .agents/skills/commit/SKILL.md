# Commit スキル

ステージ済みまたは未ステージの変更を読み取り、日本語で一行コミットメッセージを生成してコミットする。

## 手順

1. `git status` と `git diff`（ステージ済み・未ステージ両方）を確認する
2. 変更がなければ「コミットする変更がありません」と伝えて終了する
3. 未ステージの変更がある場合、ユーザーに確認してからステージングする
4. 変更内容を分析し、日本語で一行コミットメッセージを生成する
5. コミットを実行し、結果を表示する

## コミットメッセージのルール

- **必ず一行**で書く。複数行は避ける
- **日本語**で記述する
- **prefix を付ける**: `feat:`, `fix:`, `refactor:`, `docs:`, `chore:`, `test:`, `style:` など（prefix 自体は英語）
- **「何を」「なぜ」が分かる**簡潔な説明にする
- 例: `feat: ユーザー認証機能を追加`, `fix: ログイン時のリダイレクトエラーを修正`

## 注意事項

- `.env`、`credentials.json` などの機密ファイルはコミットしない。見つけた場合はユーザーに警告する
- `git add -A` や `git add .` は使わず、ファイルを個別に指定してステージングする
- コミットメッセージは HEREDOC 形式で渡す:
  ```
  git commit -m "$(cat <<'EOF'
  prefix: メッセージ
  EOF
  )"
  ```
